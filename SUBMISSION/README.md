# 详情
    描述: 待提交资源，这里是Agent开发代码和主项目之间的中间地带，经由Agent确认后提交的代码将被缓存在该区域中。供人类审核或管理员Agent进行审阅。
    结构:
        Agent_name/代码包: 这里就是Agent提交的代码
        Agent_name/commit.log: 这里是Agent提交的日记。
    补充:
        管理员相关内容:
            管理员通常指的是人类或被授予管理员身份的Agent，管理员有权限可以操作主项目中的文件，但是管理员Agent对主项中非记录类模块(也就是代码)部分进行操作时，必须经过人类的同意，管理员可以对主项中的记录类模块进行记录。
            管理员Agent一般需要具备以下的功能:
                1. 代码审核，管理员Agent往往需要对整个项目有充足的理解，并在每次审核提交的代码时候，可以根据PROJECT中的资料对代码进行审核打分和反馈。反馈的内容将直接写在各个Agent对应的操作日志中。
                2. 冲突汇报，当管理员Agent发现提交区的Agent代码存在冲突时，管理员Agent会告知发生冲突的Agent，并退回相关的代码，并告知各自的Agent应该解决的部分在哪，以此降低编写重复模块的情况。
                3. 代码通过和驳回，管理员Agent可以通过提交的代码，并将提交区域的代码移除，更新主项目对应的区域，并更新版本。当驳回时候，移除提交区域的代码，并告知Agent哪里需要修改。
                4. 一般，管理员不直接参与代码的编写。
