# 结构
├── AGENT/{Agent Name}/
│   ├── MEMORY/             # agent的记忆,agent可以调度该目录下的所有资源，回忆关于自己的所有事情。
│   ├── OPERATION_LOG/      # agent的操作日志,一般来说Agent只需获取最近若干天的操作记录即可。
│   └── SANDBOX/            # agent的沙盒环境 Agent的所有代码开发都应该放置在该资源下。

# 详情
AGENT(或类似于的内容): 
    描述: Agent资源，包括了Agent的记忆，操作记录和安全的沙盒环境，以提供Agent跨任务的超长周期项目开发支持。每个Agent都应该独立的维护好自己所在的环境。Agent所属的环境为 Agent资源/AgentName/
    结构: 
        记忆: (默认:Agent_name/MEMORY.md): 记忆是Agent进行项目理解和自身调整与学习的重要依据。Agent将开启类人类的注意力筛选机制
            Agent的记忆至少包括以下的内容:
            1. 供自己理解的最小项目概念与模块关系
            2. 项目进行中关键的纠正、缘由和反思得到的操作与实际反馈
            3. Agent进行项目开发的自我理解与核心代码的抽象概念。

        操作日记: (默认:Agent_name/OPERATION_LOG): 这里是Agent在进行任意操作时(包括但不限于对话、任务领取、Agent思考、代码提交，环境修改...)必须产生记录的区域。在每次Agent产生操作后，该区域都应该留下相关的操作记录，以确保Agent操作一致性。
            这是Agent检查自身环境和资源的重要依据，一方面，人类可以依此进行代码追踪，另外一方面，这也方便，Agent发现自身数据代码不合理的地方。
            操作日志至少包括以下内容:
            1. 操作时间
            2. 操作内容(梗概)
            3. 操作目标
            4. 操作结果

        沙盒环境: (默认:Agent_name/SANDBOX): 这是 一般Agent的主要操作环境。该沙盒往往是主项目的镜像资源，Agent可以在沙盒环境进行任意代码的开发。
            沙盒环境的获取: 沙盒环境往往由于Agent从主环境中按照需拉去，Agent不拉去开发无关部分，以此维持代码的整洁性。或者由人类手动赋值代码到沙盒环境中。但是注意，每次获取主环境的代码时，沙盒环境总会重置为最原始状态，以保持沙盒环境的整洁，当Agent想要保留数据的时候，请将相关数据移动到沙盒环境外。
            代码提交: Agent可以且仅可以将沙盒环境的代码提交到主项目中，并且Agent每次进行提交时候必须检查提交的代码，以确保备注信息的存在，并提供一条本次提交中涉及的代码信息。
    补充: Agent记忆是Agent可以持续的进行项目开发的重要依据，通过记忆模块Agent可以尽量的减少对于全局代码的依赖，减少代码检索次数。当Agent无法独立的完成相关的指标时，Agent应及时和管理员进行联系和调整。
