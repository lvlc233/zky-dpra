# 结构
├── PROJECT/
│   ├── MEMORY/             # agent的记忆,agent可以调度该目录下的所有资源，回忆关于自己的所有事情。
│   ├── OPERATION_LOG/      # agent的操作日志,一般来说Agent只需获取最近若干天的操作记录即可。
│   └── SANDBOX/            # agent的沙盒环境 Agent的所有代码开发都应该放置在该资源下。

# 详情
PROJECT(或类似于的内容): 
    描述: 项目资源，该资源描述了Agent和人类应该如何正确的开发当前项目，使项目的发展有迹可寻。是整个资源中权限等级最高，不允许普通的，未经许可的Agent对该资源进行任何的操作。否则系统或人类将会选择移除该Agent以确保项目的稳定进行。
    结构: PROJECT的三块核心组成部分
        规格文档(默认:SPECIFICATION.md): 规格文档是项目的基础，它将指导Agent以怎么样的组织形式和写作方式进行项目的开发和团队的协助。
            当Agent自己的代码不合格或尝试与原本代码冲突时，Agent将重新阅读规格文档以确保Agent能以标准的内容进行创作。
        任务指标(默认:TASK_METRICS.md): 任务指标是项目进行阶段性演化的依据，每次进行项目模块的开发时，要求一定要有一个清晰明确的可验收的任务描述和验收指标。这里相当于Agent的任务栏，Agent只能实现和完成任务栏中未完成且属于Agent的任务，每个任务都会有相关负责人/Agent，若某块任务产生了问题，希望将调研负责人/Agent的详情。
            每个Agent能且仅能进行自己分配的任务的相关模块的开发，而不允许介入其他Agent负责的模块。
            每条任务指标都有至少有以下4个要素
            1. 任务开始时间
            2. 任务完成/废弃时间
            3. 任务负责人/Agent
            4. 任务完成状态
        项目记录(LOG.md): 该记录由系统或人类进行记录，它标记着和主项目紧密相关的进度汇报。Agent可以阅读该项目的记录来快速的了解项目的核心重点部分。
            项目记录包括至少下列的内容
            1. 每次开发的代码模块正式进入到主项目时候的关键和重要内容。
            2. 每次Agent或管理员提交的重要报告。
    补充: 项目资源是项目推进的重要依据。每个Agent都有责任确保项目资源的正确性与安全性。当项目中存在项目资源混乱或被破坏时，每个Agent有权向管理员提出警告并不参与任何的项目开发。
